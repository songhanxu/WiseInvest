## 修正：日K线不再使用实时行情 high/low 强制修正最新蜡烛柱

**文件**: `ios/WiseInvest/WiseInvest/Presentation/Market/StockDetailView.swift`

**问题**: `syncKLineWithRealtimePrice()` 在 `selectedPeriod == .d1` 时，使用 `stock.high`/`stock.low`（来自 getStockQuote API 的全天高低价）覆盖日K线最新蜡烛柱的 high/low，导致K线跳变（两个不同数据源的高低值不一致）。

**修正**: 移除日K线的特殊分支，统一所有周期的逻辑——仅通过 `close` 价格自然扩展 high/low，不再从 `stock.high`/`stock.low` 强制赋值。Volume 同理，保持K线数据原值。确保日K线完全以后台K线API返回的数据为准。

## 修复：K线图双指手势与外层 ScrollView 冲突导致页面无法滚动

**文件**: `ios/WiseInvest/WiseInvest/Presentation/Components/KLineChartView.swift`

**问题**: 在 K 线图区域进行双指缩放/平移操作后，个股详情页的 ScrollView 的 `isScrollEnabled` 被永久设为 `false`，导致滚动条消失且页面无法上下滑动。

**根本原因**:
1. `wasScrollEnabled` 在每次 `disableParentScroll()` 调用时被覆盖——当 pinch 和 twoFingerPan 同时触发时，第二次调用保存的是已经被禁用的 `false`，恢复时恢复到 `false`
2. `twoFingerRefCount` 引用计数在手势状态异常时（如 iOS 跳过 `.ended` 直接到 `.cancelled`）可能永远无法归零
3. watchdog timer 的 0.8s 在持续手势操作时反复被重置，手势结束后可能来不及触发就被新手势取消

**修正**:
- **KLineGestureView**: 改用 `scrollDisableCount` 计数模式，仅在首次禁用（count 0→1）时保存原始 ScrollView 状态，避免被覆盖。增加 `forceRestoreScroll()` 方法和 `deferredRestoreTimer`（0.15s）作为最终安全网。`touchesEnded`/`touchesCancelled` 中无条件恢复。pinch/twoFingerPan 结束时通过检查另一个手势状态判断是否清除 `isTwoFingerActive` 标志。
- **PriceAxisDragView**: 同步修复，使用 `isDisabled` 标志防止重复保存原始状态，增加 `touchesEnded`/`touchesCancelled` 安全网和 `deferredRestoreTimer`。

## 修复：K线图右侧30px间距不应是永久死区

**文件**: `ios/WiseInvest/WiseInvest/Presentation/Components/KLineChartView.swift`

**问题**: 为了清楚显示当前价格和最新K线，最右边的蜡烛距右边界留了30px (`chartRightPadding`)。但这30px是一个永久不可绘制K线的死区——即使用户往右滑动，这块区域也永远空白，不会被K线填充。

**修正**:
- **移除固定布局分割**: 将 `chartRightPadding` 从布局中移除，蜡烛绘制区域 (`candleAreaWidth`) 现在覆盖整个图表宽度
- **引入动态像素偏移**: 在 `KLineChartState` 中新增 `pixelRightMargin` 属性（初始值30px），所有蜡烛绘制、价格标签、十字线的 x 坐标都减去此偏移量
- **两层滚动模型**: 手势 `handlePan` 中实现两层处理：
  - 第一层（像素级）：当 `scrollOffset == 0` 时，向右滑动消耗 `pixelRightMargin`（30→0），向左滑动恢复（0→30）
  - 第二层（蜡烛级）：`pixelRightMargin` 耗尽/恢复完后，正常增减 `scrollOffset`
- 效果：初始显示时最新蜡烛离右边界30px（保留当前价格线效果），用户向右滑动可自然填充这30px区域

## 新增：K线图惯性滑动（Momentum Scrolling）

**文件**: `ios/WiseInvest/WiseInvest/Presentation/Components/KLineChartView.swift`

**需求**: 用户快速滑动K线后松手，K线应根据滑动速度继续平滑减速滚动，而非立即停止。

**实现**:
- **`KLineChartState` 新增惯性引擎**:
  - 使用 `CADisplayLink` 以屏幕刷新率（60fps）驱动每帧减速更新
  - `momentumVelocity`: 当前惯性速度（px/s）
  - `decelerationRate = 0.96`: 每帧衰减因子（60fps归一化），模拟自然减速
  - `minimumVelocity = 15`: 速度低于此阈值时停止动画
  - `startMomentum()`: 启动惯性动画，缓存 chartWidth/dataCount/回调
  - `stopMomentum()`: 停止 DisplayLink 并清零速度
  - 碰到边界（scrollOffset clamp 到 0 或 maxOffset）时自动停止
- **`applyPixelDelta()` 共享方法**: 提取两层滚动逻辑为公共方法，被 `handlePan`（直接触摸）和 `momentumTick`（惯性动画）共用
- **`handlePan` 修改**:
  - `.began`: 调用 `stopMomentum()` 停止任何正在进行的惯性动画
  - `.changed`: 改为调用 `applyPixelDelta()` 复用逻辑
  - `.ended`: 通过 `gesture.velocity(in:)` 获取手指抬起时的速度，调用 `startMomentum()` 启动惯性
- `reset()` 和 `deinit` 中调用 `stopMomentum()` 确保资源清理
