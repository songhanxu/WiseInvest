### 自选列表"一闪而过"然后变空的竞态条件修复
**现象**：在搜索页添加自选后关闭搜索框，自选列表中一闪而过刚添加的项目，然后又变成空列表
**根因**：竞态条件（Race Condition）
1. `addToWatchlist` 发起 POST 请求（异步），同时乐观更新了 `watchlist`
2. 关闭搜索框 → `.onChange(of: showSearch)` 立刻触发 `loadWatchlist()` 发起 GET 请求
3. GET 请求先于 POST 完成 → 后端数据库中还没有新记录 → 返回空列表
4. 空列表覆盖乐观更新 → "一闪而过"现象

**iOS 端修复**（`MarketDetailViewModel.swift`）：
- 新增 `pendingMutations` 计数器，追踪正在进行中的 add/remove 请求数量
- `addToWatchlist` 和 `removeFromWatchlist` 调用时 `pendingMutations += 1`，回调完成时 `-= 1`
- `loadWatchlist()` 检测到 `pendingMutations > 0` 时，延迟 1 秒再发起 GET 请求，确保 POST 已持久化
- 拆分为 `loadWatchlist()`（带延迟保护）和 `forceLoadWatchlist()`（直接加载）

**后端修复**（`stock_handler.go` `GetWatchlist`）：
- 在返回 `stocks` 前增加 `nil` → `[]StockResponse{}` 保护，确保 JSON 永远返回 `[]` 而非 `null`

### 竞态策略升级：精确匹配 → 高水位（High Watermark）
**改动原因**：原先 `capturedSeq == currentSeq` 只接受"最后一个请求"的结果，如果发了 5、6、7 三个请求，6 先回来也会被丢弃（因为 currentSeq 已经是 7）。改为高水位机制后，只要比当前已采纳的数据更新就接受。

**新机制**：
- 每个数据源维护两个值：`counter`（递增分配）+ `highWaterMark`（已采纳的最高序号）
- 发请求前：`counter += 1`，捕获 `capturedSeq = counter`
- 回调中：`guard capturedSeq > highWaterMark`，大于则采纳并提升水位，否则丢弃
- **移除** `isRefreshingX` 去重标志 — 允许多个请求同时在飞，谁新谁生效

**涉及文件**：
- `MarketDetailViewModel.swift`：`indicesSeq` → `indicesCounter` + `indicesHighWaterMark`，`watchlistSeq` → `watchlistCounter` + `watchlistHighWaterMark`
- `StockDetailView.swift`：`quoteSeq` + `isRefreshingQuote` → `quoteCounter` + `quoteHighWaterMark`

**示例场景**：当前展示的是 seq=4 的数据，发出了 5、6、7 三个请求：
- 6 先回来 → 6 > 4 ✅ 采纳，水位提升到 6
- 5 后回来 → 5 > 6 ❌ 丢弃（比已展示的旧）
- 7 最后回来 → 7 > 6 ✅ 采纳，水位提升到 7

### K线图第九轮修复：双指操作后页面内 ScrollView 无法滚动

**问题**：用户在 K 线图区域双指缩放/平移后，StockDetailView 内部的 ScrollView 无法上下滚动
**根因**：`disableParentScroll()` 设置 `UIScrollView.isScrollEnabled = false`，但在某些时序下（尤其是 pinch 和 two-finger pan 同时识别时），gesture recognizer 的 `.ended`/`.cancelled` 回调可能不被完美触发，导致 `enableParentScroll()` 未执行。虽然之前添加了 `touchesEnded`/`touchesCancelled` 安全网，但 UIKit 手势识别器一旦识别成功会"吞掉"触摸事件，`touchesEnded` 不一定传递到 view 层

**修复方案：看门狗定时器（Watchdog Timer）**：
- 在 `KLineGestureView` 和 `PriceAxisDragView` 中各添加 `scrollRestoreTimer: Timer?`
- `disableParentScroll()` 时启动 0.8 秒看门狗定时器
- 手势 `.changed` 阶段持续刷新看门狗超时（用户还在操作就不触发）
- `enableParentScroll()` 时取消看门狗（正常恢复路径）
- 如果 0.8 秒内没有任何 `.changed` 事件且 `enableParentScroll()` 也未被调用，看门狗到期后强制重置所有状态并恢复 ScrollView

**编译验证**：KLineChartView.swift 零 linter 错误 ✅

### K线数据加载优化：后端指数退避重试 + 前端自动重试 + 错误弹窗

**问题**：某些时间刻度的 K 线一直显示骨架屏闪烁，既没有提示也没有显示数据。后端无重试机制（上游 API 一次失败就返回空数组），前端收到空数据也不主动重试。

**后端改造** (`stock_handler.go` `GetKLineData`)：
- 指数退避重试：最多 3 次重试，间隔 500ms → 1s → 2s
- 每次重试 5 秒超时，整体请求 15 秒超时
- 所有重试失败后，返回结构化错误响应 `{"error": "获取数据超时", "data": [], "retries": 3}`
- 新增 `math` 包导入

**前端改造** (`StockDataService.swift`)：
- `getKLineData` / `loadMoreKLineData` 返回 `KLineResult`（`points` + `errorMessage`）
- 新增 `KLineErrorResponse` 解析后端结构化错误

**前端改造** (`StockDetailView.swift`)：
- 客户端自动重试：最多 3 次，指数退避 2s → 4s → 8s
- 新增 `performKLineLoad()` / `handleKLineLoadFailure()` 方法
- 所有重试失败后弹窗显示后端错误消息
- 弹窗"重试"按钮重置重试计数，"取消"按钮关闭骨架屏显示空状态
- K 线图区域新增空状态 UI（图标 + "暂无K线数据" + "点击重试"按钮）
- 切换周期时重置重试状态

**编译验证**：stock_handler.go + StockDataService.swift + StockDetailView.swift 均零 linter 错误 ✅
